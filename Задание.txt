В качестве тестового задания необходимо разработать backend с API для приложения онлайн-заказов на фреймворке CodeIgniter 3.

Обязательные сущности (использовать MySQL или MariaDB):
1. Номенклатура. Информация о товарах и услугах с иерархией. Обязательные поля: наименование, группа (папка), цена, описание, URL изображения в низком и высоком разрешении.
2. Заказ. Информация о конкретном заказе клиента. Обязательные поля шапки заказа: дата и номер заказа, состояние (новый, подтвержден, оплачен, отправлен, закрыт, отменен), имя клиента, номер телефона, комментарий. Строки заказа должны содержать информацию о выбранной клиентом номенклатуре с количеством.

Должны быть реализованы следующие методы JSON REST API:
1. Получить список товаров в группе.
2. Получить подробную информацию о товаре по идентификатору (включая описание и URL изображения в высоком разрешении).
3. Создать заказ. В составе заказа передается информация о клиенте и строки заказа.
4. Получить заказ по его идентификатору.
5. Отменить заказ.

Классический сценарий использования:
1. Получаем список товаров в группе один или несколько раз.
2. Получаем подробную информацию об одном или нескольких товарах.
3. Создаем заказ на основании информации из п. 1 и 2.
4. Получаем информацию о созданном в п. 3 заказе.
5. Отменяем заказ.

Бонусные задачи (не обязательны, но их выполнение будет плюсом):
1. Несколько изображений для товара.
2. Регистрация и авторизация клиента по номеру телефона и паролю.

3. Документация на API в формате OpenAPI.




Выполнять ли по простейшему варианту или расширенному определяете из своей оценки сроков выполнения. Если понимаете, что не успеете реализовать всё - лучше реализуйте какую-то часть задания хорошо, чем весь объем, но плохо.
